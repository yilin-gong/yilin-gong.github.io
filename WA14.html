At first, I will definitely use Jupiter notebook with markdown to explain my argument. This type will be specifically for the audience who familiar with code or want to modify the parameter or function to fit with their own dataset. Also, this presenting way allows the audience to check on the raw data, code to find whether the code or result contains any bias or not. For example, the result is made based on a single episode or whole episode.

In addition, I will use the revealJS or other web tool to visualization the results. Since sometimes the code is messy especially contains some functions, it might not easily be understood by people who don’t have code experienced before, and it might destroying the reading experience because the cell has input and output, which can’t provide a fluent reading environment. However, by using the web, I can only focus on the layout of the outputs. Also, the audiences can see the result immediately after clicking on the web instead of running for several hours.

Last, I will consider using data comics. From what I noticed, more people like to watch fun stuff or fewer words such as TikTok， rather than long lecture stuff. Therefore, using data comics, allows any type of people, no matter their ages or gender, to accept and comprehend the results. since it has the storyboard to the narrative.
